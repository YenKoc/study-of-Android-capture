- 这里主要是看HTTP图解以及肉丝的博客进行学习总结，还需要计算机网络的知识，我计网基础也不咋的233，还得继续补补，总体来说，差不多明白了，https本质就是http协议+认证+ssl+完整性保护，对http明文容易被窃取以及通信方无法认证的痛点，进行改进了，同时目前的通信一般都是混合式的进行加密，先通过非对称加密对对称加密的密钥进行加密，双方得到对称加密的密钥后，就开始用对称加密了，毕竟对称加密效率高2333，大体过程见目录下的另一张图。
- 至于charles下加入证书实际上是中间人抓包的部分，因为在中间人抓包的过程中，通信过程实际上被charles切割成了两部分，分为客户端->charles,charles->服务端，两个部分charles分别担任了服务端和客户端的角色，那么就会出现一个问题，charles作为服务端时，app和手机并不信任charles的公开密钥，因为没有charles的服务端证书，所以我们需要将charles的证书下载放入系统根目录下，这样如果客户端没有ssl ping的检查的话，就可以直接过了，就算有也可以通过hook检查的api，进行bypass，问题不大，接下来就是charles作为客户端的角色了，这时候出现charles无法信任服务器端返回的公开密钥了，因为charles并没有服务端上的证书，所以我们需要将客户端的证书都装到charles上，这样就可以正常通信了。
